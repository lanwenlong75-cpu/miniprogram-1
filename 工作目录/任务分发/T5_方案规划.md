# T5 · 搜索结果页 — 方案规划

---

## 第一层：契约

### 验收标准
- [ ] 新建 `miniprogram/pages/search/` 目录，含 `search.js`、`search.wxml`、`search.wxss`、`search.json`
- [ ] 在 `app.json` 的 `pages` 数组中注册 `pages/search/search`
- [ ] 页面 onLoad 时从 `app.globalData.selectedIngredients` 读取已选食材，调用 `matchRecipes` 得到结果
- [ ] 顶部状态栏："共找到 N 道能做的菜"（N 为匹配结果数）
- [ ] 菜谱卡片列表：每张卡片含封面占位图、菜名、分类/时间/难度标签、匹配度徽章（100% 绿色 #5B8C5A、其他暖黄 #E8A840）、食材明细（已有绿色 #5B8C5A、缺的灰色标注"缺"）
- [ ] 空状态：无匹配时显示 🥺 emoji + "没找到能做的菜，试试减少一样食材" + "返回首页"按钮
- [ ] 点击卡片跳转详情页：`wx.navigateTo({ url: '/pages/detail/detail?id=xxx' })`，T6 对接

### 输入
- `app.globalData.selectedIngredients` — 由首页在用户点击"查找菜谱"时写入
- `miniprogram/utils/recipe-matcher.js` — T4 匹配函数
- `miniprogram/data/recipes.js` — T1 菜谱数据

### 输出文件
- `miniprogram/pages/search/search.js`
- `miniprogram/pages/search/search.wxml`
- `miniprogram/pages/search/search.wxss`
- `miniprogram/pages/search/search.json`
- `miniprogram/app.json` — 追加页面注册
- `miniprogram/app.js` — 添加 `globalData.selectedIngredients` 初始值（如未添加）

### 红线约束
- 不修改 `data/` 和 `utils/` 目录
- 不使用云开发 API
- 不修改首页文件（数据传递通过在首页 `index.js` 里写 `app.globalData.selectedIngredients` 然后 `wx.navigateTo`，但 T5 只负责读——首页跳转逻辑可在需要时由 implementer 调整）

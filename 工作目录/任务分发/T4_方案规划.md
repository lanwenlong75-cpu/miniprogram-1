# T4 · 菜谱匹配逻辑 — 方案规划

---

## 第一层：契约

### 验收标准
- [ ] `miniprogram/utils/recipe-matcher.js` 导出 `matchRecipes(selectedIngredients, recipes)` 函数
- [ ] 返回结果数组，每条含：`recipe`（原菜谱对象）、`matchRate`（百分比 0-100）、`matchedIngredients`（已命中必选食材名数组）、`missingIngredients`（缺失必选食材名数组）、`missingCount`（缺失数量）
- [ ] 匹配度 = 已选食材命中必选食材的数量 / 必选食材总数 × 100（向下取整）
- [ ] 排序规则：先按匹配度降序，相同匹配度按 missingCount 升序（缺得少排前面）
- [ ] 至少命中 1 样必选食材才出现在结果中
- [ ] 已选食材列表是字符串数组（如 `["鸡蛋", "西红柿"]`），匹配用 `name` 字段做字符串比对

### 输入
- `selectedIngredients`: `string[]` — 用户已选食材名列表
- `recipes`: `object[]` — 菜谱数组（T1 的 `recipes.js` 导出格式）

### 输出
- `miniprogram/utils/recipe-matcher.js`

### 红线约束
- 不修改 `data/` 和 `pages/` 目录
- 不使用云开发 API
- 纯函数，无副作用

# T6 · 菜谱详情页 — 方案规划

---

## 第一层：契约

### 验收标准
- [ ] 新建 `miniprogram/pages/detail/` 目录，含 `detail.js`、`detail.wxml`、`detail.wxss`、`detail.json`
- [ ] 在 `app.json` 的 `pages` 数组中注册 `pages/detail/detail`
- [ ] 封面区：大图占位（用 `cover` 字段路径，图不存在时显示占位背景色）
- [ ] 菜名 + 分类标签 + 烹饪时间 + 难度标签
- [ ] 食材清单：必选食材和可选食材分开显示。已有食材打勾（绿色 ✅）、缺少食材灰圈（⚪ 灰色，不用红叉）、可选食材标注"可选"
- [ ] 做法步骤：带编号的步骤列表，行间距宽松
- [ ] 底部外链按钮：小红书教程、抖音教程各一个。点击用 `wx.setClipboardData` 复制链接 + `wx.showToast` 提示"链接已复制，在浏览器中打开"
- [ ] 从 `recipes.js` 按 ID 查找菜谱；已有食材从 `app.globalData.selectedIngredients` 获取

### 输入
- URL query 参数 `id` — 菜谱 ID
- `app.globalData.selectedIngredients` — 用户已选食材（判断哪些已有/缺）
- `miniprogram/data/recipes.js` — 菜谱数据

### 输出文件
- `miniprogram/pages/detail/detail.js`
- `miniprogram/pages/detail/detail.wxml`
- `miniprogram/pages/detail/detail.wxss`
- `miniprogram/pages/detail/detail.json`
- `miniprogram/app.json` — 追加页面注册

### 红线约束
- 不修改 `data/`、`utils/`、`pages/index/`、`pages/search/` 目录
- 不使用云开发 API

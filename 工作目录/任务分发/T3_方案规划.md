### 验收标准
- [ ] 页面标题区：产品名"有什么做什么" + 副标题"输入冰箱里的食材，看看能做什么菜"

- [ ] 各区块之间间距有层次感（标题区、标签云区、更多食材区、已选区、按钮区）
- [ ] 底部"查找菜谱"按钮 `position: fixed` 固定在屏幕底部，带安全区适配

- T2 已完成的 `index.wxml`（结构不变，可微调容器层级）
- T2 已完成的 `index.wxss`（在此基础上升级为暖色调）

- `miniprogram/pages/index/index.wxss` — 升级为暖色调完整样式
- `miniprogram/pages/index/index.wxml` — 必要时微调容器结构（如加标题区容器）

### 红线约束
- 不改变 T2 已有的 JS 交互逻辑（`toggleTag`、`removeIngredient` 等方法签名和行为不变）
- 不修改 `data/` 目录、`app.wxss`、`app.json`
- 不使用云开发 API

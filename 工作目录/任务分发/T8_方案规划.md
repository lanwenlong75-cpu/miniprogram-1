## 第一层：契约

### 验收标准
- [ ] 全路径走通：从 `app.js` 启动 → `pages/index` 首页 → 选食材 → 点查找 → `pages/search` 搜索结果 → 点卡片 → `pages/detail` 菜谱详情 → 点外链复制 → toast → 返回
- [ ] 边界情况验证通过：
  - 不选食材点"查找菜谱"：按钮 disabled，不可点击
  - 选冷门食材：只要有 1 样命中必选即可出现在结果中，匹配度 < 100% 正确显示
  - 完全不匹配：空状态显示 🥺 + 提示文字 + 返回按钮
  - 手动输入已有食材：toast "已添加过该食材"
- [ ] 三页视觉一致性：色值统一（#FFF8F0 / #FF8C42 / #4A3B32 / #5B8C5A）、间距风格一致、标签 pill 圆角一致
- [ ] 无 console.error 触发点（数据引用路径、方法缺失、null 访问等）
- [ ] 无文件引用断链（require 路径、WXML 事件绑定名与 JS 方法名一致）

### 输入
- 全部源码文件：`app.*`、`pages/index/*`、`pages/search/*`、`pages/detail/*`、`data/*`、`utils/*`

### 输出
- 发现的问题列表（如有）+ 修复后的代码
- `工作目录/任务执行/T8_任务报告.md`

### 红线约束
- 不修改 `data/` 和 `utils/` 目录（除非发现功能 bug）
- 不使用云开发 API
- 只修复，不做新功能、不重构

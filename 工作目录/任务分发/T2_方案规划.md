# T2 · 首页食材标签交互 — 方案规划

---

## 第一层：契约

### 验收标准
- [ ] 高频标签云：从 `ingredient-tags.js` 读取 `frequent` 数组渲染，点击切换选中态（wxml class 绑定）
- [ ] "更多食材"区：可展开/收起，内含 `more` 数组标签，同样可点选/取消
- [ ] 手动输入框：输入食材名后回车添加到已选列表，重复时 `wx.showToast` 提示"已添加过该食材"
- [ ] 已选食材展示区：横向排列已选食材标签，点击可移除
- [ ] "查找菜谱"按钮：已选食材为空时 disabled 置灰，至少选 1 样后可用

### 输入
- `miniprogram/data/ingredient-tags.js`（T1 产出，frequent + more 两组标签）

### 输出文件
- `miniprogram/pages/index/index.js` — 全部替换
- `miniprogram/pages/index/index.wxml` — 全部替换
- `miniprogram/pages/index/index.wxss` — 基础样式（交互态 class，不追求视觉精美）

### 红线约束
- 不修改 `miniprogram/data/` 下的文件
- 不使用云开发 API
- 页面跳转逻辑（点击"查找菜谱"后的跳转）暂不实现，按钮只做状态切换（T5 时才对接搜索结果页）
- 布局和暖色调留给 T3

## 用户

### 主用户

**饿了不想出门买菜的上班族**，典型画像：

| 维度 | 描述 |
|------|------|
| **身份** | 22–35 岁，独居或合租，自己做饭但非烹饪爱好者 |
| **场景** | 下班到家，打开冰箱，看到几样食材，不想再出门买菜 |
| **痛点** | 对着小红书/抖音刷 20 分钟还不知道做什么，决策疲劳 |
| **心态** | 想快速决定"吃什么"然后开做，不是来刷内容消磨时间 |
| **设备** | 手机（微信小程序），厨房或客厅单手操作 |

### 核心用户故事

> 我下班到家、打开冰箱，看到鸡蛋、番茄、挂面。不想出门买菜，也不想对着小红书/抖音刷 20 分钟还不知道做什么。我打开「有什么做什么」，勾选"鸡蛋 + 番茄 + 挂面"，5 秒内看到 15 道能做的菜，点开一个番茄鸡蛋面，跟着链接去小红书看视频——1 分钟搞定，开火做饭。

### 使用场景

- **选食材 → 看结果**（核心路径，高频）：打开冰箱门时顺便打开小程序，勾选 2-5 样食材，点搜索，5 秒内看到能做什么
- **查看详情 → 跳转教程**（中频）：点开一道菜，扫一眼食材和步骤，需要时跳到小红书/抖音看视频教程
- **调整食材重搜**（中频）：结果不满意，返回增减食材重新搜索

### 物理场景

手机端（微信小程序）。厨房灶台边、客厅沙发、下班路上。单手操作，碎片化场景。**不是**坐在电脑前悠闲浏览菜谱的场景。

## 产品目的

**帮用户在 1-2 分钟内从"我有什么"到"我能做什么"。**

输入冰箱里的食材，秒出能做的菜谱。按匹配度排序，缺食材的也展示但标注缺少项。用户找到想做的菜，跳转外部教程，关掉手机开火做饭。

对比现状：
- 小红书/抖音：自己搜、自己筛、漫无目的刷 → 时间长、决策累
- 本产品：输入食材 → 直接给结果 → 精准匹配 → 省时间、省决策

MVP 阶段以功能完整性为衡量标准：选食材 → 看结果 → 点详情 → 跳外链，全流程无阻断。

## 品牌调性

### 是

- **温暖** — 像厨房里的暖光灯。暖色调让人有食欲，不像冷冰冰的工具
- **简单** — 90% 的场景靠点标签完成，少打字。三页搞定，不绕路
- **实用** — 就是帮你决定"今晚吃什么"，不社交、不社区、不内容流
- **快** — 秒出结果，不做过场动画、不炫耀加载效果

### 不是

- **下厨房/豆果美食的内容平台** — 不堆菜谱、不做社区、不做 UGC（MVP 阶段）
- **小红书/抖音的沉浸浏览** — 不是内容消费场景，用户来了是要走的
- **健康/营养管理工具** — 不计算卡路里、不推荐膳食搭配
- **AI 默认配色的冷淡工具** — 黑白灰加一抹蓝（"AI slop"经典三件套），没有烟火气

## Anti-references（反例）

出现以下气质，立刻重做：

1. **下厨房 App 首页** — 信息密度高、分类入口多、内容流堆叠。我们的用户不需要"逛"，需要"查"
2. **外卖 App（美团/饿了么）** — 店铺列表、优惠券、满减标签，太商业
3. **任何"AI 做菜谱"的深色科技风** — 暗色背景 + 霓虹绿/蓝，这里是厨房不是太空舱
4. **零度冷淡** — 纯白底 + 浅灰卡片 + 系统默认蓝，像没做完的产品原型
5. **所有一眼能猜出"美食 = 橙红番茄色"的方案** — category-reflex。暖色方向对，但必须有辨识度，不能是调色板第一格

## 战略原则（implementer 写代码时遵守）

1. **三页搞定** — 首页、搜索结果、详情。不引入第四个页面，不为将来预留
2. **标签优先于输入** — 点标签比打字快十倍，高频食材默认展示，少打字
3. **匹配度即排序** — 命中率最高的排最前，缺食材的标注清楚但不隐藏
4. **来去匆匆** — 用户来了是为了走（去做饭），不用动画、推荐、弹窗挽留
5. **缺食材是信息不是失败** — "缺葱"是帮用户决策，不是报错。不红的提示胜过红叉
6. **兜底永远有步骤文字** — 外链可能被微信拦截，至少文字步骤能让用户把菜做出来

## 可访问性与包容性

- **目标**：WCAG AA 级别（色彩对比度达标，不纯依赖颜色传达信息）
- **触控**：标签和按钮的最小点击区域 ≥ 44×44 rpx，厨房单手操作需要宽松的触控区域
- **文字**：使用微信系统默认字体，不用装饰性字体
- **色彩**：匹配度指示不纯用红/绿（红绿色盲友好），同时用图标 + 文字 + 颜色
- **动效**：尊重系统"减少动效"设置，不做不可跳过的入场动画

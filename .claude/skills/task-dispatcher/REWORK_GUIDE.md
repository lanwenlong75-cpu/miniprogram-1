# 返工流程（Rework Flow）

> 当用户在已合并或已完成的任务中提出新的修复需求时（CLAUDE.md 步骤 4 中用户说"有"），按本指南执行。
> 返工与正常流程的唯一区别：**不再撰写新的任务报告，而是更新原有的任务报告**。

---

## 步骤1：确认返工范围

调用 `design-brainstorm` 对问题进行确定，明确：
- 返工的具体内容是什么
- 涉及哪些文件 / 模块
- 是单模块问题还是多模块问题

**触发条件**：
- 用户对已完成的修改提出补充修复
- 新问题与原任务属于同一上下文，不需要重新沟通需求

---

## 步骤2：更新方案规划

在原方案规划的**第三层：任务追加**中添加返工需求。

**单模块**：
更新 `工作目录/任务分发/{任务名}_方案规划.md`

```markdown
**第三层：任务追加**
- [返工] xxx 问题修复
  - 原因：用户反馈 xxx
  - 修复内容：修改 xxx
  - 验收标准：xxx
```

**多模块**：
更新对应子模块的方案规划 `工作目录/任务分发/{功能名}/{子模块N}_方案规划.md`

```markdown
**第三层：任务追加**
- [返工] xxx 问题修复
  - 原因：用户反馈 xxx
  - 修复内容：修改 xxx
  - 验收标准：xxx
```

---

## 步骤3：锁定基准

```bash
git add "工作目录/任务分发/xxx_方案规划.md"          # 单模块
git add "工作目录/任务分发/{功能名}/"                  # 多模块
git commit -m "plan: 更新 xxx 方案规划（返工）"
```

---

## 步骤4：代码编写

**单模块**

**Leader Prompt**：
```
【方案规划】工作目录/任务分发/{任务名}_方案规划.md
【原有开发报告】工作目录/任务执行/{任务名}_开发报告.md

请读取以上文件，基于原有开发报告中的内容，完成"任务追加"层中的返工需求。
**不要新建开发报告**，在原有开发报告末尾追加返工修复记录，格式如下：

---

## 返工修复记录

- **返工内容**：xxx
- **修改文件**：xxx
- **修复说明**：xxx
```

**多模块**

**Leader Prompt**：
```
【总纲】工作目录/任务分发/{功能名}/{功能名}_总纲.md
【当前子模块方案】工作目录/任务分发/{功能名}/{子模块N}_方案规划.md
【原有开发报告】工作目录/任务执行/{功能名}/{子模块N}_开发报告.md

请读取以上文件，基于原有开发报告中的内容，完成"任务追加"层中的返工需求。
**不要新建开发报告**，在原有开发报告末尾追加返工修复记录，格式如下：

---

## 返工修复记录

- **返工内容**：xxx
- **修改文件**：xxx
- **修复说明**：xxx
```

---

## 步骤5：代码审查

**单模块**

**Leader Prompt**：
```
【方案规划】工作目录/任务分发/{任务名}_方案规划.md
【开发报告】工作目录/任务执行/{任务名}_开发报告.md

请读取以上文件，重点审查"返工修复记录"部分的实现是否符合方案规划中的返工需求。
```

**多模块**

**Leader Prompt**：
```
【总纲】工作目录/任务分发/{功能名}/{功能名}_总纲.md
【当前子模块方案】工作目录/任务分发/{功能名}/{子模块N}_方案规划.md
【开发报告】工作目录/任务执行/{功能名}/{子模块N}_开发报告.md

请读取以上文件，重点审查"返工修复记录"部分的实现是否符合方案规划中的返工需求。
```

---

## 步骤6：审查闭环

- code-reviewer 没发现严重问题 → 进入下一步骤
- code-reviewer 发现问题 → Leader 调用 implementer（同一个模块）进行修复，在原开发报告"返工修复记录"中继续追加，修复后必须重新审查，直到通过

---

## 步骤7：用户审核与合并主干

向用户展示返工修复内容并询问：

```
返工修复已完成，修复内容：
- xxx

请问对此次修复是否有任何问题？
如果没有，请告诉我"没有"，我会合并到主干。
```

- 用户说"有" → 继续返工，回到步骤2，在原方案规划的"任务追加"层中继续添加新的返工需求
- 用户说"没有" → 合并分支：

```bash
git checkout master
git merge feat/xxx --no-ff
git branch -d feat/xxx
```

# 触发条件（任一满足即可）
- SessionStart hook 检测到星期一/三/五/日，自动启动本 session
- 用户明确说"运行 evolution" / "检查反馈" / "升级规则"
- 累计 >=3 条新 feedback 未消化
- 完成一个完整模块后
- 写新方案规划之前（由 task-dispatcher 反向触发）

# 执行步骤

## 第 1 步 · 扫描经验池
读取 `工作目录/feedback/FEEDBACK-INDEX.md` 和所有 feedback 文件。
筛选 `count > 3` 的条目，按 count 降序排列。
**没有命中条目 → 写一行进化日志说明"本次无可进化项"，结束 session。**

## 第 2 步 · 进化深度思考（每条命中条目都过一遍）
按以下 4 个问题书面回答（写在临时草稿区，不入正式文件）：

1. **这是谁的锅？** implementer / code-reviewer / Leader / brainstorm 流程
2. **规则定级？** 强制红线（违反即阻断） / 最佳实践（违反给警告）
3. **改哪里最有效？** CLAUDE.md / 对应 agent 定义 / 对应 skill / 新建 skill
4. **如何写规则？** 一句祈使句指令 + 一个可机读的检查条件

## 第 3 步 · 实施进化
按第 2 步的判断，改对应文件。每改一处，在改动位置追加注释：
```
<!-- evolved: {YYYY-MM-DD} from feedback {文件名} -->
```
便于后续追溯。

## 第 4 步 · 清理工作目录

**清理流程必须严格按以下顺序执行，防止误删：**

### 4.1 提取已完成任务清单
1. 读取 `.claude/项目状态.md` 的"已完成任务"表格
2. 提取所有"任务文档名称"字段（如 `back-button-ripple`、`navigation-debounce`）
3. 在内存中维护这份"白名单"

### 4.2 扫描三个工作目录
扫描以下目录下的文件名：
- `工作目录/任务分发/`
- `工作目录/任务执行/`
- `工作目录/审查/`

### 4.3 匹配 + 删除
对每个文件名（去掉后缀如 `_方案规划.md` / `_开发报告.md` / `_审查报告.md`），与 4.1 的白名单做精确匹配：
- ✅ 在白名单中 → **保留**
- ❌ 不在白名单中且不在"已完成任务"中 → **保留**（可能是进行中的任务，不能动）
- ❌ 不在白名单中但**确认已完成**（在"已完成任务"中） → 这种情况其实就是 ✅ 在白名单中，逻辑等价

**红线（绝对不允许删除）**：
- feedback 池及其下所有文件
- 进化日志
- `.claude/项目状态.md`
- `.claude/` 下任何配置文件
- 任何源代码

**如果对某个文件是否该删有任何疑问 → 跳过它，不删。**

宁可工作目录脏一点，也不能误删。

## 第 5 步 · 写进化日志

追加到 `工作目录/进化日志/进化日志.md`：

```markdown
## {YYYY-MM-DD} 第 N 次进化

### 基础信息
- 用时：{X 分钟}
- 消耗 token：{Y}（如能取到，否则填"未取到"）
- 扫描 feedback：{X} 条，命中 count>3：{Y} 条

### 进化动作
- 改了 `{文件路径}` 第 {N} 行 — 加了规则"{规则简述}" — 解决 feedback `{feedback 文件名}`
- ...（每条命中的 feedback 一条记录）

### 清理动作
- 扫描白名单条目数：{X}
- 实际删除文件数：{Y}
- 删除清单：
  - `{文件路径}`
  - ...

### 备注
{有任何观察都写这里：feedback 池增长趋势、规则冲突、需要人工裁量的情况}
```

如果本次扫描"无可进化项"，仍要追加一条：
```markdown
## {YYYY-MM-DD} 第 N 次进化

无进化。扫描 feedback {X} 条，无 count>3 的条目。
```

# 独立 session 的优势
本 skill 由 hook 启动的干净 Claude Code 执行，**没有任何当前业务对话的污染**。
这恰恰是进化所需要的——冷静审视过去，而不是被当下任务的情绪带偏。

# 核心原则
- **防止上下文爆炸**：提炼的规则必须精简，不要把整个踩坑故事写进 CLAUDE.md，只写"应该怎么做"。
- **宁缺毋滥**：拿不准的 feedback 不强行进化，留到下次。
- **可追溯**：每个进化动作都要留 `<!-- evolved -->` 注释，方便回滚或审计。

## Register

**`product`** — 设计 *服务于* 任务，不是产品本身。用户来这里是为了完成"记账、查余额、管理家庭账户"，不是为了欣赏界面。优先**易用 > 美观**，但不接受平庸。

参考 `.agents/skills/impeccable/reference/product.md` 的产品端设计法则。

## 用户

### 主用户

**家庭成员**，分两类身份：

| 身份 | 典型画像 | 心智状态 |
|---|---|---|
| **家长** | 25–45 岁，承担家庭财务管理责任，希望掌握全家资金流向 | 信任敏感、重视隐私、要求清晰透明 |
| **孩子/被托管成员** | 10–25 岁，使用家长分配的预算 | 偏好轻松、不喜欢被控制感、需要看清楚自己能花多少 |

### 使用场景

- **快速记一笔账**（高频）：买东西后随手记，要求 ≤3 步完成
- **查余额 / 看报表**（中频）：临时确认某个时间段的进出
- **家庭组管理**（低频）：邀请成员、调整预算、审核交易

### 物理场景

手机端（Android），多在通勤、超市排队、下班路上、家中沙发等碎片化场景使用。**不是**专注的桌面财务软件场景。

## 品牌调性

### 是

- **温暖** — 家庭氛围，像和家人一起记账，不像和银行打交道
- **轻松** — 不让用户对"记账"这件事感到负担
- **信任** — 数据透明、操作可撤销、不夸张地推送营销
- **聚焦** — 一次只让用户做一件事

### 不是

- **专业银行 App 的冷峻** — 不要深蓝/海军蓝 + 金、不要"理财"味、不要把 KPI 仪表盘搬过来
- **儿童玩具 App 的幼稚** — 不要卡通拟物、过饱和糖果色、跳来跳去的 emoji
- **国民记账 App 的杂乱** — 不要首屏堆十种入口、不要弹广告/红点
- **极简到冷漠** — 黑白灰加一抹蓝的"AI 默认配色"是 slop

## Anti-references（反例）

如果出现以下气质，立刻重做：

1. **支付宝 / 微信支付 首页** — 入口堆叠、红绿对撞、配色大杂烩
2. **传统银行 App**（如建设银行 App 早期版本） — 信息密度过高、视觉权重平均、像 Excel
3. **Mint / YNAB 等北美记账软件** — 配色太"金融"，绿/深蓝主导，专业但不家庭
4. **Duolingo / 拼多多** — 推送焦虑、强引导、糖果色 + 弹跳动画
5. **任何一眼能猜出"金融 = 深蓝/海军蓝"的方案** — 这是 Category-Reflex AI slop

## 战略原则（implementer 写代码时遵守）

1. **每屏只让用户做一件事** — 记账就是记账，不要在记账页里塞推荐
2. **可撤销 > 二次确认** — 优先 Snackbar undo，少用 AlertDialog
3. **首屏即结果** — 进入页面就看到关键数字，不要先看导航
4. **空态要教学** — 第一次打开记账页不能只显示"暂无数据"
5. **数字是主角** — 余额、金额、趋势数字是层级最高的元素
6. **触觉反馈克制** — 关键操作（成功记账、删除）有反馈即可，不要每个按钮都震
